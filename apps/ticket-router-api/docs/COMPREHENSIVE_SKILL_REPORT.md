# 📊 Матрица Квалификации: Анализ применения технологий

Данный отчет мапит первоначальный запрос на технологии с тем, что было фактически реализовано в MVP "Support Assistant", и дает архитектурное обоснование.

## ✅ Что было реализовано в текущем MVP
1. **Python и библиотеки:**
   - `Pandas`: Применен для загрузки датасета, очистки текстов (удаление NaN), генерации новых фичей (Feature Engineering).
   - `Matplotlib` / `Seaborn`: Использованы в скрипте EDA для генерации графиков распределения (boxplot, countplot).
   - `Scikit-learn`: Использован для TF-IDF векторизации текста, метрик (Accuracy, Classification Report) и разделения выборки (train_test_split).
2. **Алгоритмы (Ансамбли):**
   - `CatBoost`: Использован градиентный бустинг для многоклассовой классификации (MultiClass). Достигнута точность 96.6%.
3. **Работа с данными (API, Очистка):**
   - `API`: Данные скачаны через API HuggingFace (`datasets` library).
   - `Очистка и подготовка`: Написаны функции очистки текста регулярными выражениями, формирование сырого `.csv`.

## ⏭️ Что было отложено (Архитектурное решение)
Следующие технологии **не были** реализованы в данном коде, так как с точки зрения CTO/PM внедрять их в текущую задачу было бы избыточно (Overengineering):

1. **Глубокое обучение (PyTorch, TensorFlow, CNN, RNN, LSTM, Attention):**
   - *Почему не сделали:* Классический CatBoost на TF-IDF выдал 96.6% точности. Использование нейросетей с механизмом Attention (Transformers) усложнило бы архитектуру (потребовалась бы GPU/MLX для быстрого инференса) ради прироста в 1-2%.
2. **SQL и Web Scraping:**
   - *Почему не сделали:* Мы выбрали путь использования готового (размеченного) API датасета для скорости. Скрейпинг сырых данных потребовал бы дополнительной ручной разметки (Labeling).
3. **Plotly:**
   - *Почему не сделали:* Для быстрого EDA генерация статических `.png` файлов через Seaborn подошла лучше, чем интерактивный HTML-график Plotly.

## 📌 Резолюция для Technical PM / CTO
Данный MVP доказал, что 80% бизнес-задач классификации решаются качественной подготовкой табличных данных и классическим градиентным бустингом (CatBoost/XGBoost).

Для подтверждения квалификации в оставшихся областях (Deep Learning, SQL, Web Scraping) рекомендуется инициировать отдельный Research-проект в директории `CAIO-AI-Strategy`.
